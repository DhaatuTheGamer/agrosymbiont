# Contributing to AgroSymbiont

First off, thank you for considering contributing to AgroSymbiont! It's people like you that make AgroSymbiont such a great tool for sustainable agriculture.

## Code of Conduct

By participating in this project, you are expected to uphold our [Contributor Covenant](https://www.contributor-covenant.org/).

## How Can I Contribute?

### Reporting Bugs or Requesting Features

- Ensure the bug/feature was not already reported.
- If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Submitting Pull Requests

1. Fork the Project.
2. Create your Feature Branch (`git checkout -b feature/AmazingFeature`).
3. Set up the development environment (`npm install`).
4. Ensure all unit tests pass (`npm run test`) and code builds cleanly (`npm run build`).
5. Commit your Changes (`git commit -m 'Add some AmazingFeature'`).
6. Push to the Branch (`git push origin feature/AmazingFeature`).
7. Open a Pull Request on the main repository.

## Testing

All test files are stored in the `tests/` directory at the project root, maintaining the same structure as the `src/` (or root) directory. When adding new components or hooks, please ensure to include testing coverage.
